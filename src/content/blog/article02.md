# 1. 计算机系统概述

## 从源程序到可执行文件

> 计算机硬件只能够执行 `机器语言程序`
> MAR位数 = 存储单元的个数 MDR位数 = 存储字长（每个存储单元的位数）

### 翻译程序

- 编译程序（将`高级语言程序`一次性翻译成`目标程序（汇编语言源程序 用助记符编写）`）
- 解释程序 (一条语句翻译成对应的机器目标代码，并立即执行，翻译一句执行一句。`不会产生目标程序`)

### 高级语言和可执行目标文件的转换过程

1. 预处理
2. 编译
3. 汇编 (翻译成机器语言指令 生成二进制目标文件.o)
4. 链接 (目标文件.o和函数库合并成可执行目标文件)

## 主要性能指标

1. 主频: 以Hz为单位，CPU时钟脉冲频率 /s
2. 时钟周期: $\tfrac{1}{主频}$
3. CPI: 执行一条指令所需的时钟周期数
4. CPU执行时间: $\tfrac{CPU时钟周期数}{主频} = \tfrac{指令条数 * CPI}{主频}$
5. MIPS: 每秒执行多少百万条指令 10^6
6. MFLOPS、GFLOPS 等， 相邻相差 10^3

### 透明（不可见） IR、MAR、MDR

# 2. 数据的表示和运算

## 定点数的编码表示

### 原码 补码 反码 移码

> 0的补码表示唯一
> 补码定点数运算时 符号可以作为数的一部分参加运算
> 32位补码表示整数范围$-2^{31} ~ 2^{31} - 1$

| -    | 原码 (首位符号位) | 补码     | 反码                | 移码 |
| ---- | ----------------- | -------- | ------------------- | ---- |
| 正数 | 1                 | 同原码   | 同原码              | -    |
| 负数 | 0                 | 反码 + 1 | 符号位不变 其他取反 | -    |

## 运算方法和运算电路

### 补码定点数加减运算溢出判别

- 一位符号位
  $V= A_s$ && $B_s$ && $S_s $ || $!A_s$ && $!B_s$ && $!S_s$, V = 1溢出
- 双符号位 $S_1 S_2$
  1.  $S_1 S_2 = 00$ 正数
  2.  $S_1 S_2 = 01$ 正溢出
  3.  $S_1 S_2 = 10$ 负数溢出
  4.  $S_1 S_2 = 11$ 负数
- 一位符号根据数据进位情况判断溢出 $V = C_s 异或 C_1$, V=1溢出

### C语言中的整数类型及类型转换

有符号无符号之间转换

- 有符号数用补码
- 无符号数用二进制

不同字长之间转换

- 大字长->小字长 高位截断
- 小字长->大字长 高位用符号拓展（无符号0拓展）

### 数据的存储和排列

#### 大小端存储

```c
  int i = 01 23 45 67H;
```

| -        | 0800H | 0801H | 0802H | 0803H |
| -------- | ----- | ----- | ----- | ----- |
| 大端对齐 | 01H   | 23H   | 45H   | 67H   |
| 小端对齐 | 67H   | 45H   | 23H   | 01H   |

#### 边界对齐存储

设 按字节编址，则通过填充 每个数据占1字节

## 浮点数的表示与运算

### 浮点数的规格化

- 正数 0.1xxx
- 负数 1.1xxx

### IEEE754

| -          | 符号 | 阶码 （补码 + 偏置值） | 尾数 （原码） |
| ---------- | ---- | ---------------------- | ------------- |
| 32位单精度 | 1位  | 8位                    | 23位          |
| 64位双精度 | 1位  | 11位                   | 52位          |

- 对于阶码 存储阶码之前 要加上`偏置值` $2^{阶码位数} - 1$
- 对于尾数 隐藏最高位1
- 对于基数 基数越大，范围越大，精度越低

### 浮点数的加减运算

1. 对阶 `小阶向大阶对齐` (只有尾数右移)
2. 尾数求和
3. 规格化 1.xx 形式
4. 舍入 （右规 、 对阶 时 舍入）
5. 溢出判断
6. C语言中的浮点数类型
