# 考研笔记-查找

> 平均查找长度 ASL = $\displaystyle\Sigma_{i=1}^{n}第i个元素 查找概率（\tfrac{1}{n}） * 比较次数$

## 树形查找

### 二叉排序树 BST `左 < 根 < 右`

二叉排序树的删除 ，被删除的结点 = z

1. 若z是`叶结点`，则直接删除
2. 若z有只有一棵左子树或右子树，子树代替z
3. z的直接后继/前驱 代替z

### 平衡二叉树 AVL `左右子树高度的高度差不超过1` `同时满足BST`

#### 平衡二叉树的调整 `每次调整最小的不平衡子树` ⚠️

插入(结点A)

1. LL（左孩子B 左子树 插入）A成为B的右子树，B原来的右子树作为A的左子树
2. RR（右孩子B 右子树 插入）A成为B的左子树，B原来的左子树作为A的右子树
3. LR（左孩子B 右子树 插入)
4. RL（右孩子 左子树 插入）

删除
与BST类似

### 红黑树

略

### B树 `平衡因子等于0的m路BST`

1. 所有叶结点都在同一层
2. `除根之外的非叶结点`至少有⌈m/2⌉棵子树，最少有 `⌈m/2⌉-1`个关键字

#### B树的插入 (优先从下层找)

如果插入的结点 插入后的关键字个数超过 m, 需要`分裂` 从中间位置 [m/2]的位置进行分裂，[m/2]位置关键字插入到父结点中。

#### B树的删除

- 当被删除的关键字k不在终端结点内
  可以利用k的前驱/后继k'(k'必须在终端结点中) 来代替k。转换成在终端结点内的情况
- 在终端结点内

1. 可以直接删除，关键字个数满足需求
2. 兄弟足够借 父子换位法（用被删除关键k的前驱/后继k' 插入到父元素中，父元素中夹在被中间的元素 插入到被删除的结点中）
3. 兄弟不够借 删除关键字后与左/右兄弟以及夹在中间的父结点关键字进行合并。

### B+树

一颗m阶b+树：

1. 每个分支结点最多有m棵子树
2. `非叶根`结点至少`两颗子树`，其他每个分支结点至少有[m/2]棵子树
3. `结点的关键字个数和子树个数相同`
4. 叶结点包含所有信息，其他结点起索引作用
5. `支持顺序查找`

### B树 B+树的异同

相同

- 都支持随机查找
- 都是平衡多叉树
- 都可以用于文件索引结构

| -    | 关键字个数    | 子树个数（n个关键字） | 终端结点                             |
| ---- | ------------- | --------------------- | ------------------------------------ |
| B树  | [m/2]-1 , m-1 | n + 1                 | 包含的关键字和其他结点的关键字不重复 |
| B+树 | [m/2] , m     | n                     | 叶结点包含了所有的关键字             |

## 散列表

查找时间复杂度 O(1), 与表中元素个数无关

### 构造法

1. 直接定址法
2. 除留余数法
3. 数字分析法
4. 平方取中法

### 冲突处理

#### 开放地址法 `寻找另一个空闲位置`

1. 线性探测
2. 平方探测
3. 双散列
4. 伪随机

#### 拉链法 `同位置用链表`

#### 散列查找性能分析

- 查找效率 = `散列函数` `处理冲突的方法` `装填因子(装满程度)`
- ASL 依赖于 装填因子(越大越容易冲突，反之可能性越小) 不直接依赖于元素个数n
- 聚集现象: 线性再探测法，因为处理冲突方法不当，影响ASL
