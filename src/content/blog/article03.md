## 2 进程与线程

### 易错帽子题

- 系统线程 可重复使用
- `中断向量`是`中断服务程序`的`入口地址`
- 互斥信号量 初始值为1
- int类型变量，读写时需要 互斥信号量保护更新

### 用户级、内核级线程

| -    | 用户级线程                                       | 内核级线程             |
| ---- | ------------------------------------------------ | ---------------------- |
| 切换 | 同一进程内不需要内核调度，跨进程需要内核参与调度 | 需要从用户态转到核心态 |
| -    | 由线程库管理                                     | 内核空间管理           |
| 阻塞 | 一个线程阻塞 整体阻塞                            | -                      |

### 进程通信

1. 共享存储 （可直接访问的共享空间）
2. 消息传递 （全双工）
3. 管道 （半双工）

### C语言 分配

1. 动态分配的内容 -> 堆
2. 传递参数和局部变量 -> 栈
3. 常量 -> 正文段

### 处理机调度

1. `进程调度`是操作系统必须的
2. 高级调度（作业调度）外存 -> 内存
3. 中级调度（内存调度）内存 -> （暂时不能运行）外存 -> （满足条件后）内存
4. 低级调度（进程调度）就绪队列 -> CPU

#### 调度指标

1. 周转时间 = 作业完成时间 - 作业提交时间
2. 平均周转时间 = ($\Sigma_{i=1}^n 作业_i的周转时间$)/n

#### 调度算法

注意事项：

1. 区分抢占/非抢占式
2. 区分 临界区、内核临界区（不能处理机调度）

##### 优先级调度算法

优先级的分类

1. 动态优先级
2. 静态优先级

优先级参照原则

1. I/O型进程 > 计算型进程
2. 交互型进程 > 非交互型进程

##### 高响应比优先调度算法

依次选择响应比最高的进行调度
响应比 = $\dfrac{要求服务时间 + 等待时间}{要求服务时间}$

## 内存管理

### 程序的链接与装入

链接

1. 静态链接 `运行之前装入`
2. 装入时动态链接 `边装入边链接`
3. 运行时动态链接 `需要时链接`

装入

1. 绝对装入
2. 可重定位装入（静态重定位）`适用于固定分区` 不能在内存中移动
3. 动态运行时装入 需要重定位寄存器支持，内存中的地址均为相对地址

### 连续分配与管理方式

1. 单一连续分配: 有内碎片
2. 固定分区分配
3. 动态分区分配

## 文件管理
1. UNIX将输入/输出设备视为`设备文件`
2. FCB必须连续
3. read系统调用（文件描述符fd, buf缓冲区首地址, 传送的字节数n）: 从fd指的文件处读n个字符，传送到buf指的缓冲区
4. open系统调用（文件名）
5. 注意编号，开始序号 + 长度 - 1 = 最终序号

### 文件保护
1. 访问类型 （限制对文件的访问类型）
2. 访问控制 （访问控制列表ACL 规定每个用户允许访问类型）灵活 安全性差 由系统实现

### 有结构文件的组织形式
1. 顺序文件 平均查找 $\tfrac{N}{2}$次
2. 索引文件 
3. 顺序索引文件 平均查找 $\sqrt{N}$次
4. 直接文件或散列文件

### 文件的物理结构
1. 连续分配（适用定长文件）
2. 链接分配
  - 隐式链接（只适合顺序访问）
  - 显式连接 （需要一张FAT 文件分配表，每个表项存放链接指针，下一个盘块号）
3. 索引分配（支持随机访问）
4. 混合索引（多级间接地址）

### 文件共享
1. 硬链接 （不同进程维护各自的文件描述符，共享同一个内存索引节点。维护一个链接技术 count，为0时才会删除）
2. 软连接 （类似快捷方式）
