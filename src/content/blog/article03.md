## 2 进程与线程

### 易错帽子题

- 系统线程 可重复使用
- `中断向量`是`中断服务程序`的`入口地址`
- 互斥信号量 初始值为1
- int类型变量，读写时需要 互斥信号量保护更新

### 用户级、内核级线程

| -    | 用户级线程                                       | 内核级线程             |
| ---- | ------------------------------------------------ | ---------------------- |
| 切换 | 同一进程内不需要内核调度，跨进程需要内核参与调度 | 需要从用户态转到核心态 |
| -    | 由线程库管理                                     | 内核空间管理           |
| 阻塞 | 一个线程阻塞 整体阻塞                            | -                      |

### 进程通信

1. 共享存储 （可直接访问的共享空间）
2. 消息传递 （全双工）
3. 管道 （半双工）

### C语言 分配

1. 动态分配的内容 -> 堆
2. 传递参数和局部变量 -> 栈
3. 常量 -> 正文段

### 处理机调度

1. `进程调度`是操作系统必须的
2. 高级调度（作业调度）外存 -> 内存
3. 中级调度（内存调度）内存 -> （暂时不能运行）外存 -> （满足条件后）内存
4. 低级调度（进程调度）就绪队列 -> CPU

#### 调度指标

1. 周转时间 = 作业完成时间 - 作业提交时间
2. 平均周转时间 = ($\Sigma_{i=1}^n 作业_i的周转时间$)/n

#### 调度算法

注意事项：

1. 区分抢占/非抢占式
2. 区分 临界区、内核临界区（不能处理机调度）

##### 优先级调度算法

优先级的分类

1. 动态优先级
2. 静态优先级

优先级参照原则

1. I/O型进程 > 计算型进程
2. 交互型进程 > 非交互型进程

##### 高响应比优先调度算法

依次选择响应比最高的进行调度
响应比 = $\dfrac{要求服务时间 + 等待时间}{要求服务时间}$

## 3 内存管理

### 程序的链接与装入

链接

1. 静态链接 `运行之前装入`
2. 装入时动态链接 `边装入边链接`
3. 运行时动态链接 `需要时链接`

装入

1. 绝对装入
2. 可重定位装入（静态重定位）`适用于固定分区` 不能在内存中移动
3. 动态运行时装入 需要重定位寄存器支持，内存中的地址均为相对地址

### 连续分配与管理方式

1. 单一连续分配: 有内碎片
2. 固定分区分配
3. 动态分区分配

## 4 文件管理

1. UNIX将输入/输出设备视为`设备文件`
2. FCB必须连续
3. read系统调用（文件描述符fd, buf缓冲区首地址, 传送的字节数n）: 从fd指的文件处读n个字符，传送到buf指的缓冲区
4. open系统调用（文件名）
5. 注意编号，开始序号 + 长度 - 1 = 最终序号

### 文件保护

1. 访问类型 （限制对文件的访问类型）
2. 访问控制 （访问控制列表ACL 规定每个用户允许访问类型）灵活 安全性差 由系统实现

### 有结构文件的组织形式

1. 顺序文件 平均查找 $\tfrac{N}{2}$次
2. 索引文件
3. 顺序索引文件 平均查找 $\sqrt{N}$次
4. 直接文件或散列文件

### 文件的物理结构

1. 连续分配（适用定长文件）
2. 链接分配

- 隐式链接（只适合顺序访问）
- 显式连接 （需要一张FAT 文件分配表，每个表项存放链接指针，下一个盘块号）

3. 索引分配（支持随机访问）
4. 混合索引（多级间接地址）

### 文件共享

1. 硬链接 （不同进程维护各自的文件描述符，共享同一个内存索引节点。维护一个链接技术 count，为0时才会删除）
2. 软连接 （类似快捷方式）

## 5 I/O管理

1. I/O接口（设备控制器）
2. I/O端口（CPU可以直接访问的寄存器）
3. DMA执行顺序
   1. 初始化DMA控制器并启动磁盘
   2. 从磁盘传输一块数据到内存缓冲区
   3. DMA控制器发出中断请求
   4. 执行DMA结束中断服务程序
4. 字符不可寻址。采用顺序存取方式
5. 更换设备 -> 更换驱动即可

### I/O控制方式

1. 程序直接控制（CPU轮询，CPU和I/O不能同时工作）
2. 中断驱动方式（I/O可以主动打断CPU，CPU和I/O可以同时工作，`但数据与I/O接口之间的传输必须经过CPU`）
3. DMA方式（解放CPU，I/O设备和内存之间直接传输，`仅在传送的开始和结束时，才需CPU干预`）

### I/O软件结构

I/O结构层次。从上（用户）到下（硬件）

1. 用户层I/O软件
2. 设备独立性软件 （`系统调用`翻译成`设备操作命令`）
3. 设备驱动程序 （初始化设备，隐藏设备控制器之间的差异）
4. 中断处理程序
5. 硬件

### 缓冲区

I/O设备到缓冲区时间 T
缓冲区到工作区时间 M
处理时间 C

1. 单缓冲 单个处理时间：max(C, T) + M，n个任务处理时间 n \* 单个处理时间 + C
2. 双缓冲 处理时间：max(C + M, T)，n个任务处理时间 n \* 单个处理时间 + M + C

### 设备分配与回收

设备特性：

1. 独占式使用设备
2. 分时共享使用设备
3. SPOOLing方式使用外部设备
   1. 输入输出井 （井管理
   2. 输入输出缓冲区 （预输入程序
   3. 输入输出进程 （输出程序

设备分配的数据结构：

1. 设备控制表DCT (一个设备对应一个DCT)
2. 控制器控制表COCT （从DCT中找到指针）
3. 通道控制表CHCT （一个通道可为多个设备控制器服务）
4. 系统设备表SDT(整个系统只有一张SDT，SDT里包含所有设备)

### 磁盘和固态硬盘

磁盘初始化过程

1. 低级（物理）格式化 —— 磁道划分为扇区
2. 对磁盘进行分区
3. 高级（逻辑）格式化 —— 创建文件系统
4. 操作系统的安装

系统启动过程

1. ROM引导程序
2. 磁盘引导程序
3. 分区引导程序
4. OS初始化程序

#### 磁盘调度算法

1. FCFS 先来先服务
2. SSTF 最短寻道优先
3. SCAN 扫描（电梯调度）
4. C-SCAN 循环扫描 （磁头到达最远端的请求，不需要到端点）

#### 固态硬盘 SSD

1. 以页为单位读写
2. 随机写慢，随机访问很快
3. 擦除寿命有限
4. 磨损技术

- 动态磨损均衡
- 静态磨损均衡
